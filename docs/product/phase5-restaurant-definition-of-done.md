# Phase 5 — Restaurant First Definition of Done

## 目的

AGATHON Voice Receptionは、現段階では汎用受付ではなく、飲食店専門の受付スタッフとして完成度を上げる。

## 対象スタイル

- `restaurant`: 落ち着いたレストラン向け
- `izakaya`: 居酒屋・カジュアル飲食店向け

美容室、クリニック、歯医者、ホテルなどへの横展開は、このPhaseでは対象外。

## 飲食店受付に必要な会話項目

| 項目 | 対応方針 |
| --- | --- |
| 予約日時 | 「今日の夜」「夕方」「来週」などを自然な目安として受ける |
| 人数 | 「4人で行きたい」などを予約意図として扱う |
| 名前 | 直前に名前を聞いた文脈では「田中です」から補完する |
| 電話番号 | 予約確保の最後に自然に伺う |
| 席の希望 | 個室、半個室、テーブル、カウンター、窓側、座敷を拾う |
| 子ども連れ | 可能可否とベビーカー等の案内を返す |
| アレルギー | 内容を伺い、厨房確認が必要な場合を自然に伝える |
| コース希望 | コース/飲み放題/席だけの相談を受ける |
| 駐車場 | 店舗設定の駐車場案内を返す |
| 営業時間 | 店舗設定の営業時間を返す |
| 定休日 | 店舗設定の定休日を返す |
| 支払い方法 | 店舗設定の支払い方法を返す |
| キャンセル確認 | 予約名と日時を伺い、店舗側で確認する流れにする |
| 遅刻連絡 | 予約名と遅れる目安を伺う |
| 貸切相談 | 希望日、人数、予算感を伺い、担当確認へつなげる |
| 営業電話対応 | 直接取次ぎせず、必要時のみ折り返し |
| スタッフ転送 | 自然な「担当に確認します」でつなぐ |

## 完了条件

- 飲食店の予約・FAQ・営業電話・遅刻・キャンセルに自然に返答できる。
- `restaurant` / `izakaya` の2スタイルだけが選択対象になっている。
- 受付スタッフではなく機械に聞こえる禁止表現が、受付会話に戻っていない。
- `npm test` が通る。
- `AI_HANDOFF_LEDGER.md` に変更理由、残タスク、次にやるべきことが記録されている。

# プロジェクト：受診先ナビ（徒歩圏内の病院検索アプリ・試作版）

## 概要
体調が悪いとき、選んだ症状から「何科を受診すべきか」の目安を案内し、
徒歩圏内でその診療科がある病院候補を表示するWebアプリのMVP。

**重要：これは診断アプリではない。** 病名の診断・薬の推奨は一切行わず、
診療科（内科・整形外科など）の案内までに限定している。

## 安全要件（このプロジェクトの絶対ルール）
1. **免責の明記**：トップ画面と結果画面の両方に「医師の診断に代わるものではない」旨を表示。
2. **緊急症状の最優先警告**：胸痛・呼吸困難・意識消失・片側まひ・大量出血・けいれん等が
   選ばれたら、診療科案内より先に「今すぐ救急車(119)/救急外来へ」を最優先表示する。
   救急相談 #7119 / 救急車 119 への導線を必ず置く。
3. **診断・投薬の禁止**：病名特定・薬の推奨はしない。診療科の案内まで。

これらは今後の改修でも崩してはならない。機能追加時は必ずこの3点を満たすこと。

## 技術構成
- 素のHTML / CSS / JavaScript（ビルド不要）。ブラウザで `index.html` を開けば動く。
- データ（`data.js`）とロジック（`app.js`）を分離し、ルールの精緻化をしやすくしている。

## ファイル構成
- `index.html` … 画面（症状フォーム・結果・免責）
- `styles.css` … スタイル
- `data.js`   … 症状マスタ / 診療科マスタ / ダミー病院データ
- `app.js`    … 症状→診療科の振り分けロジック、緊急判定、病院抽出、描画

## ロジックの考え方
- 症状ごとに対応診療科を優先度順で持ち、選択症状を集計してスコアの高い科を提示。
- `emergency:true` の症状が1つでもあれば、診療科案内より前に救急警告を出す。
- 年齢層が「こども」の場合は小児科を補足提案。
- 病院は推奨診療科に該当するものを距離が近い順に表示（現状は距離・営業時間ともダミー値）。

## 動かし方
ローカルで `index.html` をブラウザで開くだけ。サーバ起動が必要なら
`python3 -m http.server` などで配信してアクセス。

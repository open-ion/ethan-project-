# SETUP — イーサン × Notion 自動化 引き継ぎ書

ローカルの Claude Code で、この自動化環境をそのまま動かすためのセットアップ手順。
Web環境だと「既存レコードの上書き編集」がポリシーでロックされるため、**ローカル移行で編集＆「常に更新」を完全解禁する**のが目的。

---

## 0. 前提（もう終わっていること）
- Notion / Google Drive のデータは全部クラウド側にある。**作り直し不要**。
- このリポジトリの `CLAUDE.md` がイーサンのペルソナ。clone すればそのまま効く。
- Notion側の自動化マニュアル: 宮本OS 配下「⚙️ 読書→Notion 自動化マニュアル（MCP連携）」

## 1. Claude Code をインストール
- デスクトップアプリ（Mac/Windows）推奨。CLIなら:
  ```
  npm install -g @anthropic-ai/claude-code
  claude
  ```
- ドキュメント: https://code.claude.com/docs/en/overview

## 2. このプロジェクトを開く
- リポジトリを clone してそのフォルダで `claude` を起動（`CLAUDE.md` が読まれる）。

## 3. MCP コネクタを繋ぐ（同じアカウント）
- アプリのコネクタ設定、または CLI の `/mcp` から追加。
- **Notion**: 接続URL `https://mcp.notion.com/mcp` → OAuth許可（Web環境と同じもの）。
- **Google Drive**: 同様に接続。
- 接続時の「アクセスするページ選択」で **宮本OS 含む全ページ**（or すべて）を許可する。

## 4. 編集を「常に許可」← ここが移行の本丸
- ローカルで初めて `notion-update-page` を呼ぶと承認プロンプトが出る（Web環境では出なかったやつ）。
- **「常に許可 / Always allow」** を選ぶ → 以後ずっと編集が通る。
- これで **上書き編集 ＆「常に更新」が完全解禁**。

## 5. 動作確認
- 「三菱HCの投資ログを最新データで更新して」と投げる → **既存行が書き換われば成功**。

## 自動化パイプライン（使い方）
本/章PDFを Drive に置く → 一言投げるだけ:
- 「『〇〇』をDriveから読んで、読書メモと辞典に入れて」
- 「〇〇の△章だけ、用語を辞典に追加して」
- 「三菱HCの投資ログを最新データ（利回り・株価・ニュース）で更新して」

処理: 抽出 → 整形 → 重複チェック → 各DBへ書き込み（読書メモ／用語辞典／投資ログ）。

## 「常に更新」の組み方（ローカル解禁後）
- `/loop` で「定期的に注目銘柄の最新利回り・株価・ニュースをWeb取得 → 投資ログを更新」。
- ローカルなら**既存行の上書き更新**ができる。Web環境では**日付つきスナップショットの追記**で代替していた。
